# 許可証等様式 出力（choju_kyokasho）

フォーム「**鳥獣保護管理法許可申請**」（親）＋「**従事者情報**」（子）のデータを、札幌市の
**許可証等様式（`許可証等様式.xlsx`・5 シート）** に書き込んで出力するスタンドアロン Google Apps Script ウェブアプリです。
`gas_for_external_action/choju_intake`（取り込み：Excel → フォーム）の **逆方向（出力：フォーム → Excel）** にあたります。
本体アプリ（`gas/`）には一切手を入れません。

- **出力テンプレ**: `許可証等様式.xlsx`（Drive に Excel としてアップロードしたもの・**6 シート**：従事者名簿／許可証／交付通知書／振興局宛通知／警察宛通知／従事者証）。緑セル `FF00B050` ＝ 書き込み先。
  `従事者名簿` は緑なし＝マスター名簿（`choju_intake` と同じ幾何）。
  ※ **テンプレ側を改修したら Drive の最新版に差し替える**こと（交付通知書シート・3 通知の「許可の条件」欄はテンプレに存在しないと書けない）。
- **テンプレの保存先 URL は Script Properties に保存**し、生成画面/設定画面から **現在値の確認・変更**ができます。
- 生成物は **Drive に xlsx として保存**し、画面にリンクを表示します。

実体は単一ファイル `Combined.gs`（バナーコメントで `Code/domain/payload/build/fill/settings/relay/ui` に区切り）。

---

## 1. 何がどのシートに書かれるか

入力は **payload に同梱されたフォームデータ**（管理者権限不要。シート再取得はしない）。
起動元（編集画面・検索一覧の単一/複数選択）に依らず **`records[]`（`{id,no,items}`）統一フォーマット**で届き、
申請件数は `recordCount`（= records 数）で決まる。従事者は各 record の `items` に
`従事者情報/#<No>/…` で常時インライン展開される（旧 `context`/`list.childFormsByRow` は廃止）。
値は表示文字列（選択肢は `", "` 連結）だが、**パスが構造完全**なので種数・方法・免許まで復元する。

| シート | 枚数 | 内容 |
| --- | --- | --- |
| 従事者名簿 | **8 名/シート**（9 名以上はシート複製） | 従事者ごと 1 ブロック（種数固定配置・用具 P 列積層・免許/登録/銃器 Q–Z）。`choju_intake` の名簿幾何と同一。テンプレは物理 8 ブロック（行 5〜76）＝1 シート 8 名。9 名以上は従事者証と同じく `copyTo` でシートを複製（許可証番号はシートをまたいで通し番号）。シート名は常に `従事者名簿_1ページ目` / `従事者名簿_2ページ目`… |
| 許可証 | **法人=1 / 個人=人数分** | 期間・住所・氏名(法人名)・生年月日(代表者名)・種数・目的・区域・方法・措置・条件 |
| 従事者証 | **法人=人数分 / 個人=0** | 期間・許可証番号・法人名・住所・氏名・生年月日・種数・目的・区域・方法・条件 |
| 交付通知書 | 1 | 振興局宛通知と同一内容だが冒頭段落(B9)ぶん**記欄が全体に +1 行シフト**（F2/F3 は不動）。被許可者への**宛名**（法人=B3 法人名/B4 代表者名、個人=B4 氏名）を持ち、B9 の日付トークン `{{gge年M月D日}}` を**受付日**の和暦へ置換。許可の条件は C35 |
| 振興局宛通知 / 警察宛通知 | 各 1 | 許可番号(F2)・許可年月日(F3)・住所(C12)・氏名/法人名(C13)・許可証番号(C14〜。個人=範囲＋他N名／法人=別紙のとおり)・種数・目的・期間・区域・方法・**許可の条件(C34)** |

### 法人／個人の出し分け（確定仕様）

- **許可証**: 法人＝1 通（捕獲頭数＝**全員合計＝法人全体**）。個人＝全員分（各人に 1 通。頭数は **その人自身の分**）。
- **従事者証**: 法人＝従事者の数だけ（頭数＝**全員合計**、方法＝その従事者分）。個人＝**出さない**。
- **従事者名簿**: 法人・個人とも全員分の行を出す。**8 名で 1 シート**、9 名以上はシートを複製。各シートとも行 5 から詰め、許可証番号（E 列）はシートをまたいで通し番号（`Cho2_materializeRoster_`）。
  種数・数量の書き方は区分で異なる: **個人＝各従事者に自分の頭数**。**法人＝先頭従事者のブロックに全員合計（＝法人全体）だけを記入し `M13` に `※法人全体の数量` と注記、2 人目以降は種数欄を空**にする（記載例_法人。氏名/住所/用具/免許は各自）。
- **シート名は `<様式名>_<識別接尾語>`（複数枚になりうる 3 種は常に接尾語を付与）**: 従事者名簿＝`従事者名簿_1ページ目` / `従事者名簿_2ページ目`…（ページ番号）。許可証・従事者証＝`許可証_<氏名>` / `従事者証_<氏名>`（法人許可証は `許可証_<法人名>`）。単票のみの通知類（交付通知書・振興局・警察）は接尾語なし。
- **両通知**: 中身は同じ。頭数は **常に全員合計**（法人＝許可された捕獲数／個人＝全員での捕獲数。いずれも合算値）。
- 住所/氏名: 法人＝法人住所・法人名・代表者名／個人＝代表（先頭従事者）の住所・氏名。
  通知の許可証番号欄: 個人＝従事者証番号の範囲（C14〜E14）＋他N名（G14）／法人＝許可番号（C14）＋従事者は「別紙のとおり」（C15）。

## 2. セットアップ（初回のみ）

1. clasp 等で本プロジェクトを GAS にプッシュ（`appsscript.json` に **Advanced Drive Service v2** 依存。
   xlsx ↔ スプレッドシート変換に必要。スコープ＝spreadsheets / drive / userinfo.email）。
2. GAS エディタで `Cho2_authorize` を一度実行し、Drive / Sheets スコープに同意。
3. **テンプレ URL を登録**（どちらでも可）:
   - GAS エディタ: `Cho2_registerSettings("<許可証等様式.xlsx の Drive URL または fileId>", "<出力フォルダID(任意)>", "<共有シークレット(任意)>")`
   - または **ウェブ画面**（`/exec` を直接開く＝設定画面）でテンプレ URL を入力して保存。
4. **ウェブアプリとしてデプロイ**（アクセス: 全員 / 実行: 自分）。`/exec` URL を控える。
5. 本体アプリの対象フォーム設定 →「外部アクション」:
   - **検索一覧ボタン**: `settings.externalActions.search[]` に **URL=`<exec>`・adminOnly=OFF** で 1 つ追加（label 例「許可証等の出力」）。
     従事者（子）は **送信時に全 formLink を on-demand 取得してインライン展開する**ので
     `includeChildData` の ON/OFF に依存しない（本体 `useSearchPageState.js` の `externalActionChildFormFields`
     は includeChildData で絞らず全 formLink を対象にする）。`includeChildData` は一覧の**表示**用フラグ。
   - **単票ボタン**: レコード詳細の外部アクションボタンに同 URL を設定（単票も子を常時インラインする）。
   - `/dev` ではサーバ間リレー不可＝**`/exec` 必須**。
   - 本体に誤送信防止シークレット（`NFB_EXT_ACTION_SECRET`）を設定している場合のみ、同じ値を `CHO2_EXT_ACTION_SECRET` に登録。
6. **（任意）全ページ一括印刷の連携**: 印刷 Web アプリ `gas_for_spreadsheet/print_kyokasyo` の `/exec` URL を登録すると、
   生成結果画面に「**全ページ一括印刷**」ボタンが出る（押すと生成した許可証スプレッドシートを
   `?ssId=…&auto=1` で開き、全シートを結合した**しおり付き 1 PDF**を自動生成してブラウザ内で印刷できる）。
   - GAS エディタ: `Cho2_registerSettings(null, null, null, "<print_kyokasyo の /exec URL>")`、または設定画面の「印刷 Web アプリ URL」欄で保存。
   - **重要**: 印刷 Web アプリは `executeAs: USER_DEPLOYING` で動くため、生成された許可証スプレッドシートに
     **印刷アプリのデプロイ者がアクセスできる**必要がある。→ **本アプリ（choju）と印刷アプリを同一 Google アカウントでデプロイ**する
     （または出力フォルダを印刷アプリのデプロイ者に共有）。未登録ならボタンは非表示。

## 3. 使い方

1. 本体の検索一覧／レコード詳細で「許可証等の出力」ボタンを押す → 本体がサーバ間リレーで本アプリを叩き、
   返ってきた生成画面 URL が新しいタブで開く。
2. 生成画面で **テンプレ URL（現在値の確認・変更）** と **対象申請・従事者の内容** を確認し、
   必要なら申請者区分（個人/法人）を上書き → 「生成」。
3. テンプレを複製して緑セルにリテラル値を書き込み、**スプレッドシートを Drive に保存**してリンクを表示。
4. （印刷 Web アプリ URL を登録済みなら）「**全ページ一括印刷**」ボタンを押す → 印刷ページが別タブで開き、
   全シートを結合したしおり付き 1 PDF を自動生成。ビューアの印刷ボタン／Ctrl+P で一括印刷、または別タブ表示・ダウンロードも可。

## 5. デプロイ後 E2E で確認する点

セル番地は **実テンプレ `許可証等様式.xlsx` と記載例（`許可証等様式_記載例_個人/法人.xlsx`）に整合済み**。
GAS 実機（テンプレ複製・記入・スプレッドシート保存・リレー受信）は node では検証不可なので、出力を記載例と照合する。

- **日付**: payload は西暦 `YYYY-MM-DD`（または `/`）前提でパースし、セルには **和暦の文字列（リテラル）** を書く。
  Google スプレッドシートは和暦の数値書式 `[$-411]ggge"年"m"月"d"日"` を解釈しない（＝書式がきかない）ため、
  `Cho2_pushDate_` が build 層で `Cho2_warekiString_`（本体 `formatEraNonPadded` の移植・**元年表記**）により
  和暦文字列へ変換して書き込む。記載例どおり和暦（例 令和8年6月1日／昭和50年1月1日、初年なら 令和元年…）に
  なるか目視（セルを開くと生テキストが見える＝書式非依存）。
- **番号は素のまま書く**（セル書式が装飾する）: 許可証 `C3`＝`1-1`/`1-1-1`（書式 `第@号`）、従事者証 `C4`＝`1-1-1`（同）、
  通知 `F2`＝許可番号（書式 `札環対許可第 @ 号`）、通知 `C14`＝許可/従事者証番号（`第@号`）、通知 `G14`＝他N名（`他#名`）。
  許可番号が空なら番号欄は空のまま。`Cho2_permitNo_` で `第…号` を付けると二重装飾になるので付けない。
- **許可証**: H4/H5＝有効期間 開始/終了、G30＝方法・G32＝措置・G34＝条件。
- **従事者証**: K14＝許可番号（=「1-1」）、K16＝法人名、D17/D23/D29＝住所/氏名/生年月日、K27/K29/K31/K33＝目的/区域/方法/条件、C4＝従事者証番号。
- **振興局宛通知 / 警察宛通知**: 従来欄に加え **C34＝許可の条件**（処分の種類が「許可」で条件なしなら空のまま）。
- **交付通知書**: 記欄は振興局の **+1 行**（住所 C13・氏名/法人名 C14・許可証番号 C15〜・従事者 C16・種数 C18:G26・目的 C27・期間 C29/F29・区域 C31・方法 C33・**許可の条件 C35**）。F2/F3 は不動。宛名は 法人=B3/B4・個人=B4。
  B9 段落の `{{gge年M月D日}}` は `Cho2_applyTokenReplacements_` が**セルをクリアせず**受付日の和暦に差し替える（段落本文は温存）。シート順は 許可証→**交付通知書**→振興局→警察→従事者証。
  ※ 法人の交付通知書は「振興局と同じ法人表記」（被許可者=法人名・許可番号 1-2・従事者=別紙のとおり）で出力する（記載例_法人の個人風表記は不採用）。
- **非緑セル**（許可証 C3 / 従事者証 C4 の証番号）も生成時に上書きしている。運用上「許可後に職員記入」なら書かない方針へ変更可。
- 番号体系・様式が変わったら `domain.gs` セクション（グリッド base・`CHO2_CLEAR_RANGES_`）と各 `Cho2_*Cells_` の番地を直す。

## 6. コーディング規約

本体 `gas/` に合わせています。`var` + `function name(){}` スタイル、内部ヘルパは末尾アンダースコア、
関数接頭辞 `Cho2_` / 定数 `CHO2_`。番地は `domain.gs` セクションの緑セル表・グリッド設定に集約。
