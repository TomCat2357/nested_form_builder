# 鳥獣保護管理法様式 ↔ フォーム 双方向ブリッジ（choju_yoshiki）

フォーム「**鳥獣保護管理法許可申請**」と、札幌市の様式ワークブック（**新 7 シート**）の間を
**双方向**でつなぐスタンドアロン Google Apps Script ウェブアプリです。本体アプリ（`gas/`）には一切手を入れません。

- **書き出し（フォーム → Excel）**: レコード詳細の **外部アクション** ボタンが `payload`(JSON) を POST すると、
  様式テンプレートを複製して全シートに**リテラル値**を書き込み、生成スプレッドシートへのリンクを返します。
- **取り込み（Excel → フォーム）**: doGet の**アップロードページ**で記入済み xlsx を選ぶと、サーバ側で解析し
  `uploadRecords`(JSON) を返します。利用者はそれを本体アプリの取り込み口（管理者 > Playground 等）で
  `sync_records` に渡してレコード化します（このアプリは本体へ直接書き込みません＝越境書込みなし）。

実体は単一ファイル `Combined.gs`（バナーコメントで `Code/payload/cellmap/domain/fillExport/parseImport/setup/Test` に区切り）。

---

## 1. 色＝役割の契約（設計の核）

様式 **申請書 L6/L7/L8** にスウォッチ付きの凡例があり、これが読み取り/書き出しの契約です。
色は**設計時の注釈**で、実行時は固定セル番地（`cellmap.gs` セクション）を引きます。

| 色 | RGB | 凡例 | 取り込み | 書き出し |
| --- | --- | --- | --- | --- |
| **黄** | `FFFFFF00` | 正として吸い取る | **権威**として読む | リテラルを書く |
| **桃** | `FFEAD1DC` | 確認用に吸い取る | **照合のみ**（名簿集計の数式。源にしない） | リテラルを書く |
| **緑** | `FF00B050` | 掃き出し場所 | **無視**（出力専用） | リテラルを書く |

**数式は一切使いません**。雛形に残る数式（桃の `=SUM/UNIQUE`、緑の `=申請書!…`）は
「どのセルへ何を書くか」の**仕様書**として読み、出力は全部リテラル値です。

## 2. データ構造

- **従事者名簿が唯一のソース・オブ・トゥルース**。申請書の桃セル（申請者個人・種数・方法）は名簿の集計。
- **名簿の幾何**: ヘッダ 1〜4 行、`firstRow=5 / blockHeight=9 / blockCount=10`（行 5〜94）。**1 ブロック = 1 従事者**。
  - 列: `E`許可番号(緑/出力) / `F`住所 `G`氏名 `H`職業 `I`生年月日(黄・9 行結合) /
    `J`種類 `K`頭数 `M`第2種(卵/ノイヌ/アライグマ) `N`第2数量 /
    `P`用具(行ごとに積層) / `Q-T`狩猟免許 / `U-W`狩猟者登録 / `X-Z`銃器 / `AA`備考。`AC:AE`=入力規則の元リスト（無視）。
  - 種数は申請書と同じ**固定オフセット配置**（キジバト=行 0 …）。方法は P 列に積層。
- **個人/法人**: 個人=申請者を名簿の代表従事者から導出（桃は照合のみ）。法人=申請書 F6/F8/F10 を**正**として読む。
  - 取り込み時の判定: 申請書 F11（生年月日）が日付なら個人、空なら法人。

## 3. 主要セクション（`Combined.gs`）

| セクション | 役割 |
| --- | --- |
| `Code.gs` | `doPost`（書き出し / `mode=import`）・`doGet`（状態 + アップロードページ）・応答・アクセスキー・probe HMAC |
| `payload.gs` | `record.items` の `/` パス分解・子レコードのグルーピング・日付/数値/シリアル変換・和暦 |
| `cellmap.gs` | **全部データ**。ラベル定数・値変換表・申請書セル配置・名簿幾何・事由書/区分グリッド・緑シート投影 |
| `domain.gs` | payload → 書き出しモデル。`Cho_parseWorker_`（ライブ JSON 準拠の免許/銃器パス）・種数合算・用具和集合 |
| `fillExport.gs` | 申請書の正準グリッドを組み、名簿・事由書を書き、緑シート（許可証/通知/従事者証）へリテラルを投影 |
| `parseImport.gs` | リーダ抽象で名簿→子・申請書/事由書→親を復元し `uploadRecords` を生成（純ロジック） |
| `setup.gs` | 一次セットアップ（不要シート削除・数式消去・色付きセル値クリア。冪等） |
| `Test.gs` | GAS エディタ用の純ロジックテスト（`testAll`） |

> 番地を直すときは `cellmap.gs` セクションだけ編集すれば済みます。取り込み・書き出しは同じマップを引きます。

## 4. セットアップ（初回のみ）

1. `form_test/鳥獣保護管理法様式_個人想定.xlsx`（または用意した清書テンプレ）を Drive にアップ →
   「ファイル > Google スプレッドシートとして保存」で変換し、ファイル ID を控える。出力先フォルダ ID も控える。
2. clasp などで本プロジェクトを GAS にプッシュ（`appsscript.json` に **Advanced Drive Service v2** 依存あり。
   取り込みの xlsx→スプレッドシート変換に必要）。
3. GAS エディタで `Cho_registerSettings("<テンプレID>", "<出力フォルダID>", "<アクセスキー>")` を実行。
4. `Cho_setupCleanTemplate()` を実行（冪等。不要シート削除＋数式消去＋黄/桃/緑セルの値クリア）。
5. **ウェブアプリとしてデプロイ**（アクセス: 全員 / 実行: 自分）。`/exec` URL を控える。
6. 本体アプリ側: 申請フォームに formLink「従事者情報」があること（子データは payload に自動同梱）。
   - 書き出し: 外部アクション 質問カードの URL に `<exec>?k=<アクセスキー>`。
   - 取り込み: 様式アップロードは `<exec>?page=import` を開く。

## 5. どのシートに何が書かれるか（書き出し）

| シート | 役割 | 内容 |
| --- | --- | --- |
| 申請書 | 正準グリッド | 申請者・種数(固定配置)・目的・期間・区域・方法(リテラル)・処置・区分○ |
| 従事者名簿 | マスタ | 従事者ごと 1 ブロック（種数固定配置・用具積層・免許/登録/銃器） |
| 事由書 | 申請事由 | 被害原因の鳥獣○・被害者(自身/以外+住所/氏名)・時期・区域・理由・備考 |
| 許可証 / 振興局宛通知 / 警察宛通知 | 出力(緑) | 申請書セルをリテラル投影。**許可番号・許可年月日・文書番号は空**（許可後に職員が記入） |
| 従事者証 | 出力(緑) | 代表従事者分（許可証番号は空） |

> ルール: フォーム由来の値はリテラルで書き、**許可後に決まる欄は空のまま**残す。

## 8. フォーム側の整合（様式に合わせて修正済み）・要確認

- **免許/登録番号**: フォームの `番号` は「**〇〇第xxxx号**」一本（旧 `番号接頭語` フィールドは全削除）。
  Excel 名簿 `S`/`V` 列の値と**そのまま一致**するため接頭語の分解/結合は不要（`Combined.gs` から撤去済み）。
- **被害の内容**: 親フォームの該当 message（ラベルは現在 **`証明書`**。Excel の対応シート名は `事由書`）に
  `被害の内容` が追加され、Excel `事由書!E26` と双方向で対応。
- **規則7条の区分**: フォームの選択肢を様式の 11 区分ラベルに揃えたので ○ グリッドと **1:1**（ロッシーなし）。
  `CHO_AREA7_MARK_` 参照。`具体的名称` は 特定猟具使用禁止区域/特定猟具使用制限区域/休猟区/猟区 に保持。
- 数式変換: 取り込みは**常に名簿マスタを直接読む**ので、桃の集計（`TEXTJOIN/UNIQUE` 等）が変換で崩れても影響なし。
- 再取り込み: アップロード時に**親レコード ID**を指定すると既存を上書き更新（子は `<親ID>_cN` で安定）。未指定は新規。
- 本番投入前に、実際に記入された xlsx と本体 Playground の実 payload で番地・パスを突き合わせること。

## 9. コーディング規約

本体 `gas/` に合わせています。`var` + `function name(){}` スタイル、内部ヘルパは末尾アンダースコア、
関数接頭辞 `Cho_`、定数 `CHO_`。`isDisplayed`/構造 HTML は doGet の自前ページのみ（本体の単一 HTML 配信とは別経路）。
