# 苦情・通報の取り込み（kujo_intake）

札幌市 CMS「**お問い合わせフォーム**」の CSV を取り込み、フォーム
「**R8環境共生担当課_苦情・通報等対応一覧**」の **Data シートへ直接書き込む**スタンドアロン
Google Apps Script ウェブアプリです。本体アプリ（`gas/`）には一切手を入れません（`choju_yoshiki/` の兄弟）。

- **CSV ファイルアップロード**（Shift-JIS / UTF-8 自動判定）→ ブラウザがデコード → GAS が**列ヘッダ駆動**でマッピング。
  **1 データ行 = 1 問い合わせ = 1 レコード**。
- **重複は取り込まない**（§1）。重複判定には**問い合わせ日（日時）も含める**＝内容も日時も一致する行だけを重複として除外。
- **最大 30 行**（重複・空行除外後）をプレビューし、**取り込む行をチェックボックスで選択**できる。
- **直接書き込み**: choju と同方式で、選択した行を対象フォームの Data シート（12 行目以降）へ append する。
  書き込み先スプレッドシートは**検索画面の外部アクションボタン押下時にリレーで自動取得**（Script Property はフォールバック）。
- **外部 AI / API 不使用**: CSV のデコードはブラウザの `TextDecoder`（完全クライアント内）。Gemini 等は使いません。
- **分類は様式駆動 + 人手レビュー**: CSV 列ヘッダごとの固定マッピングだけ自動化。相談大分類・対象種・相談種類など
  **ネストの深い意味分類は自動では入りません**。取り込み後にシート/本体側で人手補完してください。

実体は単一ファイル `Combined.gs`（バナーコメントで `Code/codec/schema/text/csv/mapper/upload/api/write/ctx/date/setup/Test` に区切り）と、
取り込み画面 `Index.html`（HtmlService テンプレート）。

## 取り込み画面の開きかた（2 通り）

1. **検索一覧の外部アクションボタン**（例「末端問い合わせ csv 取り込み」）。本体 GAS がサーバ間リレーで
   この `doPost` を叩くと、`doPost` は親フォームの保存先（`storage.spreadsheetId` / `sheetName`）を **ctx トークン**に退避し、
   **取り込み画面の URL（`?page=import&ctx=<token>`）を `openUrl` で返す**。本体フロントがそれを新しいタブで開く（choju と同方式）。
   - 外部アクションの **URL は本アプリの `/exec`**（パラメータ不要）。`/dev` は所有者専用なので**必ず `/exec`** を貼ること。
2. **URL を直接開く**: `<exec>?page=import`（ctx 無し。この場合は Script Property の書き込み先を使う）。

> 本送信の前に本体が **誤送信防止プローブ**（`nfbProbe=1`）を送ることがあり、その場合は共有シークレットの HMAC 署名を返す（§3）。
> シークレット未設定なら直接送信（既定）。

## 1. しくみ

1. `doGet(?page=import&ctx=...)` で `Index.html` を配信。画面上部に**書き込み先スプレッドシート URL**を表示（`Kuj_getTargets_`）。
2. **CSV ファイル選択**（複数可）→ [取り込み内容を確認]。ブラウザが `file.arrayBuffer()` で読み、**文字コードを自動判定**してテキスト化（`TextDecoder`）。
   判定順: ①UTF-8 BOM → UTF-8、②`fatal:true` の UTF-8 デコードが通れば UTF-8、③通らなければ Shift-JIS。
3. `google.script.run.Kuj_previewCsv_(csvTexts, ctx)` → GAS が **ピュア CSV パース**（`Kuj_parseCsv_`）→
   **列ヘッダ駆動マッピング**（`Kuj_csvRowToCandidate_`）→ **ファイル横断 dedup（時間込み）＋空行除外 → 先頭 30 行**（`Kuj_parseCsvBatch_`）→
   **読みやすいラベル＝値プレビュー**（`Kuj_previewRowsFromCandidates_`）。
4. 取り込む行に**チェック**を入れて [選択分をスプレッドシートへ取り込む] → `google.script.run.Kuj_commitImport_(csvTexts, 選択index, ctx)`。
   GAS が**同じ CSV を再パース**（ブラウザ送信値は信用しない）→ `Kuj_resolveTargets_` で書き込み先解決 →
   選択 index の各レコードを `Kuj_withLock_` ＋ `Kuj_appendRow_` で **Data シートへ直接 append**（NFB ヘッダ 11 行・データ 12 行目以降）。
   結果に「取り込み先スプレッドシートを開く」リンクを返す。

### 重複の取り込み防止

次の **13 列がすべて一致する行は同一とみなし、最初の 1 件だけ取り込みます**（`KUJ_CSV_DEDUP_COLS_`）:
`問い合わせ日 / 現在の振分先 / 問い合わせ件名 / メールアドレス / 氏名 / ふりがな / 年齢 / 職業 / 住所 / 郵便番号 / 電話番号 / 件名 / 内容`。
**`問い合わせ日`（日時）を含める**ので、内容が同じでも**日時が違えば別レコード**として残します（同じ CSV を二度取り込んだ等の
**完全一致だけ**を重複として除外）。**`ステータス`・`返信者` は判定対象外**。

- **ファイル内・ファイル横断**: `Kuj_parseCsvBatch_` が `Kuj_csvDedupKey_`（上記 13 列を trim して連結したキー）で 1 バッチ内をまとめて除外。
- **バッチ内で完結**: この外部アクションは**書き込み先シートの既存行とは突合しません**（シートは件名+内容→相談詳細、ふりがな/年齢/職業→備考
  のようにマージ保存するため、生 CSV 列ベースの dedup キーを既存行から復元できない）。
  **同じ CSV を後日もう一度取り込むと二重登録される**ので注意（1 回の取り込みで完結させる運用）。

### 30 行制限

重複・空行を除いたうえで**先頭 30 行**（`KUJ_MAX_ROWS_`）だけを対象にします。超過分は警告を出すので、
残りは CSV を分割して別途取り込んでください。

### 様式駆動マッピング（`Kuj_csvRowToCandidate_`）

| 入力 | 自動で入る項目 |
| --- | --- |
| **お問い合わせフォーム CSV**（札幌市 CMS） | `問合せ方法`=ホームページ（固定）, `受付日`=問い合わせ日（canonical 化）, `問合せ元`=氏名, `問合せ元　連絡先`=メール/電話/郵便番号/住所（「, 」連結）, `相談詳細`=(件名\|\|問い合わせ件名)+内容, `担当者`=返信者, `備考`=ふりがな/年齢/職業（あるものをラベル付きで） |

> **自動化しない / 人手で補完**: `相談大分類` と配下ツリー（`/野生鳥獣/対象種`・`/相談種類`・`/回答（簡易）` 等）、
> `現場住所等の区`、`継続/完結`、`タグ` など。書き込み後にシート/本体側で補完してください。

### enum の唯一の真実源

`KUJ_OPTIONS_`（選択肢ラベル）と `KUJ_FIELDS_`（プロパティ ↔ data キー）を、**mapper の enum 防御**で共有します。
**フォームの選択肢を変えたら `KUJ_OPTIONS_`（必要なら `KUJ_FIELDS_`）を更新**してください。未知ラベルは破棄＋警告（壊れず劣化）。

### 列の照合（直接書き込み）

data キーは `/` 連結ラベルパス（可逆エスケープ）。書き込み時はシートのヘッダ 1〜11 行から列キー（`Sheets_pathKey_`）を作り、
data キーと**同一コーデックで照合**して列に値を置きます（`Kuj_buildNewRow_` / `Kuj_appendRow_`）。
`受付日`（`YYYY-MM-DD`）は日付型に、数式接頭辞（`=` 等）は中和（先頭 `'`）して書きます。ヘッダに無いキーは取り込まれず警告に出ます。

## 2. 文字化け対策

- 一部フォント由来で「長→⾧」「氏→⽒」のように **Kangxi/CJK 部首コードポイント**が混ざることがあります。
  `Kuj_normalizeText_` が該当範囲（U+2E80–U+2EFF, U+2F00–U+2FDF）を **NFKC で通常の漢字に復元**します（全角！？（）等の表記は保持）。
- Shift-JIS デコードは**ブラウザの `TextDecoder`** が担い、GAS はデコード済みテキストを処理します。

## 3. セットアップ

1. **clasp push**: このフォルダを `rootDir` にして新規 GAS プロジェクトへプッシュ（`Combined.gs` + `Index.html` + `appsscript.json`）。
   - `appsscript.json` のスコープは `spreadsheets`（Data シートへ書き込み）+ `userinfo.email`（`createdBy`/`modifiedBy` 用）。
   - ⚠️ スコープ変更で再認証が要るのは**このツールを使う管理者本人のみ**。本体アプリのユーザーには影響しません。
2. **設定登録**: GAS エディタで

   ```js
   Kuj_registerSettings("<formId 任意>", "<書き込み先スプレッドシートID（任意・リレーで自動取得するなら空）>", "", "", "")
   ```

   を実行（`Kuj_checkSetup()` で確認）。引数は `(formId, spreadsheetId, sheetName, accessKey, extActionSecret)`。
   - `spreadsheetId` は**フォールバック**。検索画面の外部アクションボタンから開けばリレーの ctx で自動取得されるため、通常は空で可。
     直接 `?page=import`（ctx 無し）で開く運用にするなら登録しておく。
   - `sheetName` は省略時 `Data`。
   - `accessKey` を入れると取り込み画面に `?k=<accessKey>` を要求（任意・既定 OFF）。
   - `extActionSecret` は**誤送信防止プローブの共有シークレット**（任意）。本体 GAS の `NFB_EXT_ACTION_SECRET` と**同じ値**を入れると、
     本体は本送信の前に nonce プローブで宛先を検証する。空（既定）なら本体はプローブを送らず直接送信（後方互換）。引数省略時は既存値を据え置く。
3. **ウェブアプリとしてデプロイ**（アクセス: 全員 / 実行: 自分）。`/exec` URL を控える。
4. **検索一覧の外部アクションを設定**: 親フォームの「外部アクション（検索）」にラベル（例「末端問い合わせ csv 取り込み」）と
   **URL = 本アプリの `/exec`**（パラメータ不要）を登録し、`管理者のみ` に。
   ボタン押下 → `doPost` が `openUrl=<exec>?page=import&ctx=...` を返す → 取り込み画面が新しいタブで開く。
5. 取り込み画面で **CSV ファイル選択** → [取り込み内容を確認] → チェックで選択 → [選択分をスプレッドシートへ取り込む]。

## 5. リスクと対処

| リスク | 対処 |
| --- | --- |
| 書き込み先が未解決（ctx 無しで直接開いた等） | `Kuj_registerSettings` で `spreadsheetId` を登録（フォールバック）。画面上部に未解決警告を表示。 |
| 同じ CSV を後日再取り込み＝二重登録 | dedup はバッチ内で完結（§1）。1 回の取り込みで完結させる運用にする。 |
| CSV ヘッダが様式と不一致 | ベストエフォートで可能な列のみ取り込み、警告を表示。 |
| 30 行を超える CSV | 先頭 30 行のみ対象＋警告。残りは CSV を分割して取り込む。 |
| ネスト分類は自動化しない＝精度は人依存 | プレビューでラベル＝値を確認し、取り込み後に補完。 |
| enum ドリフト（選択肢変更） | `KUJ_OPTIONS_` 一元管理。未知ラベルは破棄＋警告。 |
| スコープ変更による再認証 | 影響は**このツールを使う管理者本人のみ**。本体ユーザーには波及しない。 |

## 6. コーディング規約

本体 `gas/` に合わせています。`var` + `function name(){}` スタイル、内部ヘルパは末尾アンダースコア、
関数接頭辞 `Kuj_`（移植した NFB レイアウト系は `Sheets_` / `Nfb_`）、定数 `KUJ_` / `NFB_`。
構造 HTML は doGet の自前ページ（`renderHtml_`）と `Index.html`（HtmlService テンプレート）のみ（本体の単一 HTML 配信とは別経路）。
非 BMP 文字（絵文字等）は HtmlService 配信で壊れるため使わない。
