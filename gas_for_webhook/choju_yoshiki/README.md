# 鳥獣保護管理法様式 生成 Webhook（choju_yoshiki）

フォーム「**鳥獣保護管理法許可申請**」のレコードから、札幌市の様式ワークブック
（`form_data/鳥獣保護管理法様式.xlsx` を Google スプレッドシート化したもの）を
**自動で埋めて出力する**スタンドアロン Google Apps Script ウェブアプリです。

本体アプリ（`gas/`）には一切手を入れません。本体のレコード詳細画面にある
**webhook ボタン**を押すと、隠しフォーム POST でこのウェブアプリに `payload`（JSON）が送られ、
テンプレートを複製して全シートに値を書き込み、生成したスプレッドシートへのリンクを返します。

---

## 1. 何をするものか

```
[本体アプリ] レコード詳細の webhook ボタン
      │  隠しフォーム POST (payload = レコード全項目の JSON)
      ▼
[このウェブアプリ] doPost
      │  1. payload を解析しドメインモデルを組み立て
      │  2. テンプレートのスプレッドシートを Drive 上で複製
      │  3. 全シート（申請書 / 従事者名簿 / 依頼書 / 許可伺書 …）に値を書き込み
      ▼
[応答ページ] 生成したスプレッドシートへのリンク + 警告一覧
```

### 設計の前提（重要）

- **数式は全廃**。テンプレートからは入力専用シート（`Sheet1`・`申請内容`）を削除し、
  残るシートの**数式セルをすべて消去**したうえで、このアプリが**全シートに直接値を書き込みます**。
  - 旧 xlsx の数式（例: `申請書!G6 = '申請内容'!B2`）が「どのセルに何を書くか」の仕様書になっており、
    それを `mapping.gs` のセルマップに転記しています。
- **従事者名簿は「従事者 × 捕獲方法」で 1 ブロック（6 行）**。
  - ある従事者が 3 つの捕獲方法を持つなら 3 ブロックを占有します。
  - 2 ブロック目以降は、氏名・住所・職業・生年月日・種数など**捕獲方法に関係ない欄は空白**にします
    （数量の二重計上を防ぐため）。
  - **わな（はこわな＋くくりわな）は免許が同一なので 1 ブロック**にまとめ、P 列のサブスロットに用具を並べます。
  - **銃（散弾銃＋ライフル銃）は所持許可番号が銃ごとに異なるので 2 ブロック**に分けます。
- 出力先は**テンプレートのコピー（スプレッドシート）**。職員が中身を確認・手直し（許可番号等）してから印刷する運用を想定。

## 2. ファイル構成

| ファイル | 役割 |
| --- | --- |
| `Code.gs` | ウェブアプリのエントリ（`doPost` / `doGet`）・応答 HTML・アクセスキー照合 |
| `payload.gs` | `record.items` の索引化、`/` パス分解、`従事者情報/#N/...` 子レコードのグルーピング、日付/数値変換 |
| `domain.gs` | payload → ドメインモデル構築（申請者合成・種数・**従事者×捕獲方法の名簿展開**） |
| `mapping.gs` | **全部データ**。シート別セルマップ・種数テーブル定義・名簿ブロック幾何・値変換表・ラベル定数 |
| `fill.gs` | テンプレート複製と全シート書き込み（静的マップ / 種数テーブル / 名簿ブロック） |
| `setup.gs` | 一次セットアップ（テンプレ清掃・Script Properties 登録）。GAS エディタから手動実行 |
| `Test.gs` | デプロイ不要テスト（モデル展開・フィル・異常系）。`testAll` を実行 |
| `TestPayload.gs` | テスト用ゴールデン payload（xlsx 原本の作例を再現） |
| `appsscript.json` | スコープ（spreadsheets / drive / userinfo.email）とウェブアプリ公開設定 |
| `scripts/extract_formula_map.py` | xlsx から数式・結合・表示形式を TSV 抽出する開発補助（一回限り、再実行可） |

> **セル番地を直したいときは `mapping.gs` だけ**を編集すれば済むようにしてあります（ロジックは触らない）。

## 3. セットアップ手順（初回のみ）

### 3-1. テンプレートを準備する

1. `form_data/鳥獣保護管理法様式.xlsx` を Google ドライブにアップロードする。
2. そのファイルを開き、**[ファイル] > [Google スプレッドシートとして保存]** で変換する。
   - 結合セル・和暦などの表示形式・印刷設定が保持されているか目視で確認する。
3. 変換後のスプレッドシートの**ファイル ID**を控える（URL の `/d/<ここ>/edit`）。
4. 出力先フォルダを Drive に作り、その**フォルダ ID**を控える。

### 3-2. GAS プロジェクトを作る

- 方法 A（推奨・clasp）:
  ```powershell
  cd gas_for_webhook/choju_yoshiki
  npx --yes @google/clasp create --type webapp --title "鳥獣保護管理法様式 生成"
  # → .clasp.json が作られる（rootDir は "." に。これは gitignore 済みなのでコミットされない）
  npx --yes @google/clasp push
  ```
- 方法 B（手貼り）: GAS エディタで空のスタンドアロンプロジェクトを作り、`*.gs` と `appsscript.json` を貼り付ける。

### 3-3. 設定値を登録し、テンプレートを清掃する

GAS エディタで以下を順に実行する。

1. `Cho_registerSettings("<テンプレファイルID>", "<出力フォルダID>", "<アクセスキー>")` を実行
   （引数を直接書き換えて実行。`<アクセスキー>` は任意の合言葉。空文字ならゲート無効）。
2. `Cho_setupCleanTemplate()` を実行
   （`Sheet1`・`申請内容` を削除し、全シートの数式セルを消去。**冪等**なので何度実行してもよい）。
   - 実行ログにシートごとの消去セル数が出る。`scripts/out/formulas.tsv` の件数と突き合わせて消し漏れがないか確認する。

### 3-4. ウェブアプリとしてデプロイする

GAS エディタの **[デプロイ] > [新しいデプロイ] > [ウェブアプリ]** で:

- **アクセスできるユーザー: 全員（匿名ユーザーを含む）**
- **次のユーザーとして実行: 自分**

> ⚠️ 「全員（匿名含む）」が**必須**です。隠しフォーム POST にログインリダイレクトが挟まると
> POST 本文が失われ（リダイレクト後は GET になる）、別アカウント・未ログインのユーザーで壊れます。
> 代わりに `?k=<アクセスキー>` による軽量ゲートで保護します。

デプロイ後に出る **`/exec` URL** を控える。

### 3-5. 本体アプリ側を設定する（コード変更なし）

1. フォーム編集画面に formLink「**従事者情報**」が設定されていることを確認する（特別な設定は不要）。
   - レコード詳細の webhook ボタンから送る payload には、**子フォームの行（従事者の子レコード）が自動で含まれます**。
     formLink を持つ親フォームは常に子データを payload へ展開します（旧版にあった `includeChildData` 設定は廃止）。
2. 申請フォームに **webhook の質問カード**を追加する。
   - ラベル例: 「Excel 様式を作成」
   - URL: `<デプロイの /exec URL>?k=<アクセスキー>`
   - 管理者限定（adminOnly）: **OFF**（職員全員が押せるように。受信側は storage を使わない）

## 4. 使い方（運用）

1. 本体アプリで「鳥獣保護管理法許可申請」のレコードを開く。
2. 「Excel 様式を作成」ボタンを押す。
3. 新しいタブに成功ページが開き、生成されたスプレッドシートへのリンクが表示される。
   - 申請者・個人/法人・従事者数（名簿ブロック数）が一覧表示される。
   - 枠超過などがあれば**警告**が黄色で表示される。
4. リンク先のスプレッドシートを開き、必要に応じて許可番号・許可年月日などを手直しして印刷・提出する。

生成ファイル名は `鳥獣保護管理法様式_<レコードNo>_<yyyyMMdd_HHmmss>` で、設定した出力フォルダに作られます。

## 5. どのシートが・どんな条件で埋まるか

| シート | 条件 | 主な内容 |
| --- | --- | --- |
| 申請書 | 常時 | 申請者・種数・目的・期間・区域・方法・処置・備考。申請日は送信日時の JST 日付 |
| 従事者名簿 | 個人申請のとき | 従事者×捕獲方法のブロック展開（最大 7 ブロック） |
| 従事者名簿 (法人) | 法人申請のとき | 同上（最大 34 ブロック） |
| 証明書 | 「証明書又は依頼書の別」=証明書 | 被害発生の時期/区域/内容/対策/実績 |
| 依頼書 | 同 =依頼書 | 依頼者住所・氏名・期間・被害状況・依頼理由 |
| 許可証個人 / 許可証法人 | 個人 / 法人 | 許可番号・許可年月日（許可処分情報があれば）・条件 |
| 許可伺書・交付通知書・従事者証・許可審査表・振興局宛通知・警察宛通知・報告書添付 | 常時 | レコード由来のセルのみ。決裁日・文書番号など事後データは空欄 |
| 結果報告書 | 常時 | ヘッダのみ（捕獲実績は事後入力のため空欄） |
| わな | わな（はこわな/くくりわな）使用時 | 申請者・期間・区域等 |

> ルール: **旧数式がレコード由来データに解決されるセルは書く / 許可後にしか決まらないデータは空欄のまま残す**。

## 6. テスト

GAS エディタで `testAll`（`Test.gs`）を実行し、実行ログを見る。

| テスト | 内容 |
| --- | --- |
| `testModel_golden` | ゴールデン payload のモデル組み立て（申請者合成・名簿展開・各種テキスト）を assert |
| `testModel_multiMethod` | 「わな猟＋空気銃＋銃2丁」の従事者が 4 ブロックに展開され、2 番目以降の個人欄が空になることを assert |
| `testFill_golden` | `doPost` を通して実際に様式を 1 枚生成（**テンプレ設定済みのときのみ**。未設定なら SKIP） |
| `testDoPost_missingPayload` / `testDoPost_badJson` | 異常系のエラーページ |

**ゴールデンケース**は xlsx 原本に入っていた作例（秋はじめ／春元負比呂、キツネ 10 頭、くくりわな、依頼書）を再現したものです。
生成シートの `申請書!G8`・`従事者名簿!E5`（第8-81-1号）などを原本と見比べて確認できます。

> モデル層・フィル層は純粋なロジックなので、ローカルの Node でも GAS API をスタブして検証できます（実装時に検証済み）。

## 7. payload の実測（本番投入前に必ず）

`TestPayload.gs` のゴールデン payload は、`builder/src/features/preview/printDocument.js` の
出力仕様に合わせて**手組み**したものです。本番に入れる前に、本体アプリの
**Playground（管理者 > Playground > Webhook モード）** で実レコードの payload を取得し、次を突き合わせてください。

- `record.items[].question` の実際のパス形（特に子レコード `従事者情報/#1/...`）
- **日付値の文字列書式**（`"YYYY-MM-DD"` を想定。違えば `payload.gs` の `Cho_toDateOrText_` を調整）
- チェックボックス値の `", "`（カンマ＋空白）連結
- 子フォームの方法ラベル `捕獲等又は採取等の方法（使用する捕獲用具の名称)` の**閉じ括弧が半角**であること
  （親フォームは全角閉じ＋ラベル内空白。`mapping.gs` のラベル定数 `CHO_L_CHILD_METHOD_` / `CHO_L_PARENT_METHOD_` に隔離済み）

差異があれば `mapping.gs`（ラベル/番地）か `payload.gs`（書式変換）を直すだけで対応できます。

## 8. 開発補助スクリプト（任意）

セル番地は xlsx の旧数式が典拠です。番地を見直したいときは:

```powershell
cd gas_for_webhook/choju_yoshiki/scripts
python extract_formula_map.py
```

`scripts/out/` に次が出力されます（`.gitignore` 済み・再生成可能）。

- `formulas.tsv` … シート名・セル番地・数式（転記対象セルの機械列挙）
- `merges.tsv` … 結合レンジ一覧（左上番地の検証用）
- `number_formats.tsv` … 日付セル等の表示形式（Date 書き込み先の確認用）
- `plain_values.tsv` … 固定文言・原本サンプル値

## 9. コーディング規約

本体 `gas/` に合わせています。

- `var` ＋ `function name() {}` スタイル（`let`/`const`/arrow を持ち込まない）
- 内部ヘルパは末尾アンダースコア、関数接頭辞は `Cho_`、定数は `CHO_`
- 公開エントリ（`doPost`/`doGet`）と payload 解析・HTML レンダラは
  `gas_for_webhook/template/` の雛形を流用
